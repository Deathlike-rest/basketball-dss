# СППР — Комплексная оценка баскетболистов

Прототип системы поддержки принятия решений тренерского штаба, реализующий методику
из магистерской диссертации «Анализ методов отслеживания физической активности для
проектирования системы принятия решения сборной по баскетболу».

Система оценивает игроков на основе **атрибутного подхода**: 21 атрибут в 7 категориях
нормализуется в шкалу 25–99 и сворачивается в **позиционно-взвешенный интегральный
рейтинг** с тремя экспертными модификаторами. На его основе реализованы три сценария
поддержки решений: предложение стартовой пятёрки, индивидуальные рекомендации по
тренировкам и раннее предупреждение о падении формы (риск травмы).

## Соответствие диссертации

| Компонент | Раздел | Где в коде |
|-----------|--------|------------|
| 21 атрибут, 7 категорий | 2.2.2, табл. 1 | `backend/core/attributes.py` |
| Шкала 25–99, `A = 25 + 0,74·p(x)` | 2.2.6 | `backend/core/normalize.py` |
| Матрица весов по позициям | 2.2.4, табл. 3 | `backend/core/weights.py` |
| Интегральный рейтинг `R_P`, модификаторы `R*` | 2.2.5 | `backend/core/rating.py` |
| Сценарий 1: стартовая пятёрка | 2.4.3 | `backend/core/lineup.py` |
| Сценарий 2: рекомендации | 2.4.3 | `backend/core/recommendations.py` |
| Сценарий 3: раннее предупреждение (>1σ) | 2.4.3 | `backend/core/early_warning.py` |
| 7 сущностей БД | 2.4.2 | `backend/models.py` |
| 4 роли, bcrypt | 2.5.2, 2.5.4 | `backend/auth.py` |
| Экран тренера (4 блока), карточка игрока (5 вкладок) | 2.5.2 | `frontend/src/views/` |
| Согласованность с экспертом (ρ Спирмена) | 3.3, табл. 7 | `backend/api/ratings.py` |

Стек точно по разделам 2.5–3.1: **Flask + SQLAlchemy + NumPy/Pandas/SciPy/scikit-learn**
(сервер приложения), **Vue 3 + Chart.js** (клиент), **PostgreSQL 14** (БД), трёхуровневая
архитектура.

## Запуск через Docker (рекомендуется)

```bash
docker compose up --build
```

- Клиент: http://localhost:5173
- API: http://localhost:5000/api
- БД наполняется демо-данными автоматически при первом запуске.

## Запуск без Docker (локально, на SQLite)

```bash
# Backend
cd backend
python3 -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
export DATABASE_URL="sqlite:///dev.db"
python seed.py          # наполнение демо-данными
python app.py           # сервер на :5000

# Frontend (в отдельном терминале)
cd frontend
npm install
npm run dev             # клиент на :5173 (проксирует /api на :5000)
```

## Демо-данные (команда апробации, раздел 3.2)

14 игроков (4 PG, 3 SG, 3 SF, 2 PF, 2 C), сезон АСБ. Воспроизведены два
документированных кейса:
- **Соколов Артём** (PG): рейтинг растёт за сезон (защита периметра 47→61,
  трёхочковый 52→65) — раздел 3.6;
- **Семёнов Антон** (PF): резкое падение прыгучести и скорости перед плей-офф →
  срабатывание раннего предупреждения — раздел 3.5.

## Тесты

```bash
cd backend && python -m pytest tests/ -q
```

Проверяют контрольные точки нормализации (табл. 4), суммы весов = 1,000, диапазоны
рейтинга и модификаторов, пересчёт весов при отсутствии данных, логику раннего
предупреждения.

## Вне рамок прототипа

Полноценный нейросетевой модуль прогноза разницы очков и автоматический видеоанализ
(раздел 2.4.4) в апробации применялись минимально/экспертно (раздел 3.7); здесь они
не реализованы. `scikit-learn` остаётся в зависимостях как задел для прогнозного
модуля. API для трекинговых систем архитектурно зарезервирован (раздел 2.5.3).
